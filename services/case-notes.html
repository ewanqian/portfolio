Case Notes / 案例说明

这一页不是完整作品清单，而是用几个典型服务方向，帮助合作方理解不同项目为什么会落在不同报价区间里。

1. AI 视觉短视频：为什么适合做成长期服务

这类项目的特点是：

  • 节奏快
  • 更新频率高
  • 内容量大
  • 客户通常需要持续测试风格
  • 很适合 AI 工具进入工作流

它的价值不一定在于单条片子多贵，而在于：

  • 能不能稳定更新
  • 能不能低成本维持风格
  • 能不能让品牌有持续的内容输出

所以这类项目很适合做成：单条起步 → 系列合作 → 月度内容服务

2. 开场视觉 / 演出视觉：为什么价格更高

开场视觉和普通短视频最大的不同是：

  • 它是舞台关键段落
  • 容错率更低
  • 节奏与结构要求更高
  • 屏幕 / 空间 /现场流程限制更多
  • 往往需要更完整的版本确认

所以它通常不是按"时长"报价，而是按：项目重要性、内容复杂度、版本需求、演出语境来判断。

3. 双目 / XR 内容：为什么建议先从样片开始

很多合作方第一次接触双目、Apple Vision Pro 或 XR 项目时，最容易低估的就是：

  • 平台适配
  • 观看方式
  • 格式要求
  • 空间观看与普通视频观看的差异
  • 数字环境风格与沉浸感之间的关系

所以这类项目，通常更适合先做：测试样片 → proof of concept → 小型演示版

这样能更快知道：这个方向是否成立，预算是否匹配，要不要继续做完整版本。

4. 工程调试 / 结构整理：客户真正买的是什么

这类项目里，客户买的往往不是"产出一个视频"，而是买：

  • 你帮他减少返工
  • 你帮他看清工程问题
  • 你帮他整理 workflow
  • 你帮他把混乱内容压成更稳定的结构

所以这类服务通常更适合按小时、按次或按诊断范围来报价。

5. 数字展览 / 展示系统：为什么不能只看页面数量

很多人会以为做数字展览只是"搭几个页面"。但实际更重要的是：

  • 展品怎么分类
  • 文案怎么组织
  • 展示入口怎么建立
  • 作品之间怎么被阅读
  • 页面怎么形成完整体验
  • 展后资料怎么继续保留

所以这类项目不只是页面制作，而更像：策展 + 内容整理 + 页面系统 + 公开表达

这也是为什么它通常不会只按"做几个页面"来定价。
